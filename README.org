* About This Repository

These are my personal dotfiles for the machines and environments I use.
I manage them declaratively with [[https://github.com/yunfachi/denix][Denix]], [[https://github.com/nix-darwin/nix-darwin][nix-darwin]], and [[https://github.com/nix-community/home-manager][Home Manager]].

The repository is intentionally tailored to my hosts, accounts, preferred tools, and desktop setup.
It is not a general-purpose Nix configuration, framework, or starter template, and I do not try to preserve compatibility for other users.
The source may still be useful as a reference or a starting point for a fork.

Denix discovers =.nix= files under =hosts/=, =modules/=, and =rices/=.
The repository's ownership rules and change checklist are in =docs/denix-architecture.md=.

* Daily Use

From the repository root, I enter the development shell and use =just= for the maintained workflows:

#+begin_src sh
nix develop
just
just check
just home
just switch
#+end_src

New files used by the flake must be git-tracked before evaluation.
I use the Nix-native workflows in this repository instead of maintaining ad hoc package state.

** Deploying the =seiran-vm1= Tart VM

On a Nix-equipped VM without =darwin-rebuild=, bootstrap it with one switch:

#+begin_src sh
sudo nix run nix-darwin/master#darwin-rebuild -- switch --flake .#seiran-vm1
#+end_src

Use the installed command for subsequent switches:

#+begin_src sh
sudo darwin-rebuild switch --flake .#seiran-vm1
#+end_src

Both commands activate the nix-darwin system and its integrated Home Manager configuration.

* Repository Layout

- =hosts/= contains machine facts and host-only bindings.
- =modules/= contains reusable program, service, shared-data, and toplevel integration modules.
- =rices/= contains desktop-experience policy selected through repository-owned interfaces.
- =docs/= contains stable cross-cutting policy and runbooks.
- =packages/= contains repository-owned package derivations.
