* Overview

This repository is my declarative macOS setup built with
[[https://github.com/yunfachi/denix][Denix]], [[https://github.com/nix-darwin/nix-darwin][nix-darwin]], and
[[https://github.com/nix-community/home-manager][Home Manager]].

It is currently centered on one Apple Silicon host, =kurogane=, and uses
Denix to compose host definitions, reusable modules, and theme variants
("rices") into both user-level and system-level configurations.

What this repo optimizes for:

- reproducible workstation setup with Nix flakes
- modular configuration without hand-wired import trees
- themeable desktop and editor setup through reusable rices
- AI-heavy local tooling, including Codex, Claude Code, OpenCode, and MCP wiring
- a layout that future me can re-enter without reverse-engineering the whole tree

* Mental Model

Denix auto-discovers =.nix= files under =hosts/=, =modules/=, and =rices/=.
That gives the repo its shape:

- =hosts/= defines machine-specific facts such as platform, default rice, and host-only wiring
- =modules/= contains reusable configuration split into =config/=, =programs/=, =services/=, and =toplevel/=
- =rices/= contains theme variants and inheritance-only theme bases

The important constraints are:

- Do not manually stitch modules together across those Denix discovery trees.
- Flakes only see git-tracked files, so new files must be added before evaluation or builds.
- Rices should stay data-oriented. Modules own package resolution and platform-specific wiring.
- Secrets are consumed through =config.sops.secrets.<key>.path=, not hardcoded paths or plaintext values.

Current host/theme reality:

- Host: =kurogane=
- Platform: =aarch64-darwin=
- Default rice: =monochrome=
- Other selectable rices: =catppuccin=, =everforest=

* Daily Workflows

Apply the Home Manager layer only:

#+begin_src sh
nh home switch
#+end_src

Apply the full nix-darwin + Home Manager system for =kurogane=:

#+begin_src sh
nh darwin switch . -H kurogane -Lt
#+end_src

Switch to a different rice:

#+begin_src sh
nh darwin switch . -H kurogane-catppuccin -Lt
#+end_src

Common maintenance commands:

#+begin_src sh
nix flake check
nix fmt
nix develop
#+end_src

=nix flake check= validates the flake outputs and formatting checks.
=nix fmt= runs treefmt-based formatting for Nix, Lua, and shell files.
=nix develop= enters the repo dev shell used for local maintenance work.

* Key Subsystems

** Theming

Theme data lives in =modules/config/colorschemes/= and =rices/=.
Rices choose colors, wallpaper, and theme-oriented option values; modules
consume those values to configure tools such as Ghostty, tmux, nixvim,
SketchyBar, and JankyBorders.

The inheritance-only rice at =rices/inheritanceOnly/laptop.nix= holds shared
laptop defaults. Concrete rices such as =monochrome=, =catppuccin=, and
=everforest= build on top of it.

** Secrets

Secrets are declared centrally in =modules/toplevel/secrets.nix= and bound to
the current host in =hosts/kurogane/secrets.nix=. The encrypted source of truth
is =hosts/kurogane/secrets.yaml=, managed with
[[https://github.com/Mic92/sops-nix][sops-nix]].

If you need the rotation procedure, use =docs/secrets-key-rotation.md= rather
than expanding the root README.

** AI And Agent Tooling

This repo has first-class configuration for:

- Claude Code
- Codex
- OpenCode
- agent-browser
- shared MCP server wiring

The MCP source of truth lives in =modules/programs/mcp-servers/default.nix=.
That module owns server definitions, target-specific adapters, validation
assertions, and secret-aware runtime wiring.

Some MCP helpers come from =node-packages/=, which is a bun2nix-managed Node
package set used by the Nix wrappers in =modules/config/node-packages.nix=.

** Development Environment

The flake defines a small maintenance shell with tools such as =bun=, =deno=,
=nickel=, =stylua=, and =emmylua-ls=. Formatting is configured through
=treefmt.nix=.

This is a Nix-native repo. Prefer changing the flake and modules over managing
tool state outside the configuration, unless a local module explicitly requires
it.

* Repository Map

#+begin_example
.
├── docs/           Stable policy docs and runbooks
├── hosts/          Host-specific definitions and bindings
├── modules/        Reusable Denix modules
├── node-packages/  bun2nix-managed Node packages for MCP-related wrappers
├── nickel/         Nickel experiments and support material
├── rices/          Theme variants and inheritance-only theme bases
└── var/            Small repo-local artifacts and planning scratch files
#+end_example

Within =modules/=:

- =config/= holds shared data and cross-cutting configuration helpers
- =programs/= configures user-facing tools
- =services/= configures desktop services such as AeroSpace and SketchyBar
- =toplevel/= handles broad system/user wiring such as Nix, Home Manager, fonts, and secrets

You will also see directories such as =.kiri/=, =.agents/=, =.claude/=,
=.agent-browser/=, =.opencode/=, and =.crush/=. Those are local runtime or
tool-state directories, not the main declarative source of truth.
