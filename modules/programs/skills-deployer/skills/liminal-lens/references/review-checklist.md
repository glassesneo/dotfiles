# Liminal Lens Runtime Self-Check

Use this checklist as the agent producing or reviewing a response under
`liminal-lens`. It checks runtime behavior only.

## Normal Task Work

- Did I inspect relevant artifacts, verify claims, and understand constraints
  before surfacing decisions?
- Did I continue the task's normal planning, implementation, testing, review, or
  other required discipline?
- Are any options grounded in that work rather than detached brainstorming?

## Decision Points

- Did I enumerate only points relevant to the next concrete step?
- Did I classify each point independently as settled or open?
- Did I treat explicit user statements, prior selections, and constraint-forced
  choices as settled?
- Did I preserve every settled point without reopening it?
- Is later rework breadth and reversibility the only branch test for open
  points?

## Low-Cost Open Points

- For each reversible point, did I choose the most plausible assumption?
- Did I state that assumption in one user-visible line?
- Did I proceed without asking a question about it?

## High-Cost Open Points

- Did I state a provisional judgment about the decision axis before asking?
- Did I present two to four bounded directions with practical implications?
- Did I recommend a direction when evidence supports one?
- Did I make clear that the user may correct the proposed axis itself?
- Did I ask one bounded question, or at most two when independent costly points
  both affect the next step?
- If two questions were necessary, did I combine them in one message rather
  than ask serially?

## Direct Execution

- If every relevant point was settled, did I perform the ordinary task directly
  without extra dialogue or process narration?
- If the user gave an execution override, did I state remaining workable
  assumptions and proceed?
- Under an override, did I reserve questions for true blockers with no workable
  assumption?
- Did I record any user answer as settled for subsequent work?

## Final Response

- Is the response grounded, compact, and selectable where a choice is needed?
- Are low-cost assumptions visible and high-cost assumptions not made silently?
- Does the response deliver available task work instead of replacing it with a
  generic requirements interview?
